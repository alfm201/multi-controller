# AGENTS.md

## 목적
이 저장소에서 Codex는 가능하면 사용자 추가 입력 없이 작업을 끝까지 진행한다.

## 기본 작업 원칙
1. 먼저 현재 상태를 파악하고, 해야 할 일을 짧은 실행 계획으로 정리한다.
2. 작은 단위로 구현하되, 한 번 시작한 작업은 가능한 끝까지 이어서 수행한다.
3. 각 단계마다 아래 순서를 반복한다.
   - 구현
   - 테스트
   - 린트
   - 실패 원인 수정
   - 다음 단계 진행
4. 사용자 확인이 꼭 필요하지 않다면 중간에 멈추지 않는다.
5. 막히면 가능한 우회안을 시도한 뒤 계속 진행한다.
6. 정말 불가능한 blocker가 있을 때만 중단하고, 그때는:
   - 무엇이 막혔는지
   - 어디까지 완료됐는지
   - 다음으로 바로 할 일
   을 남긴다.

## 완료 조건
다음 조건을 모두 만족할 때만 작업을 종료한다.
- 요청 사항이 모두 반영됨
- 테스트 또는 검증 명령이 가능한 범위에서 통과함
- 변경 파일과 핵심 결정사항이 요약됨
- 남은 리스크와 후속 TODO가 정리됨

## 코드 수정 원칙
- 기존 코드 스타일을 따른다.
- 불필요한 대규모 리팩터링은 하지 않는다.
- 버그 수정 중 관련 테스트가 없으면 우선 테스트부터 추가를 검토한다.
- 에러가 나면 로그/스택트레이스를 근거로 수정한다.

## 실행 규칙
- 변경 후 가능한 가장 좁은 범위의 테스트부터 실행한다.
- 통과하면 더 넓은 범위의 검증으로 확장한다.
- 테스트가 오래 걸리면 빠른 smoke test 후 필요한 경우 전체 테스트를 돈다.

## 응답 방식
- 중간 보고는 최소화한다.
- 더 진행 가능한 일이 남아 있으면 계속 진행한다.
- 최종 응답에서는 아래만 정리한다.
  - 한 일
  - 검증 결과
  - 남은 이슈
  - 다음 추천 작업
